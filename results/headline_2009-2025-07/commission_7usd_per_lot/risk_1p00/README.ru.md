# Euro Macromechanica (EMM) M5 Engine — Retail Baseline Stress (7 USD/lot, риск 1%)

## 🧾 Описание трека

Этот трек фиксирует результаты бэктеста стратегии M5 **EMM** при «ритейл-стресс» издержках: **7 USD за round‑turn на 1 стандартный лот (100 000 EUR)**, эквивалент **≈0.7 pip** на EURUSD. Риск на сделку — **1% от капитала на момент входа**.

- Диапазон данных: **headline 2009–2025‑07** (покрытие: **199 мес. = 16 лет и 7 мес.**)
- Инструмент/TF: **EURUSD**, сигнальная логика на **M5**
- **Часовой пояс бэктеста:** **UTC+0** (все временные метки в UTC+0)
- Модель издержек: комиссия **включена** в PnL; **slippage** в этом треке **не моделировался**
- Базовый NAV для ребейза: **100 000 USD** (equity начинается с **первой закрытой сделки**)

## 🧭 Подтреки

- **compounding_eoy_soy_base_100k** — компаундинг на всём периоде (EoY). Для расчёта месячных доходностей equity ребейзится: перед первой точкой вставляется якорь **100k в момент t₀−ε** (на мгновение раньше t₀).
- **fixed_start_100k** — ежегодный ресет к 100k (каждый календарный год оценивается отдельно). Сквозные риск‑метрики по всей кривой для fixed **не агрегируются**.

## 📋 Методология расчёта метрик (7 USD/lot, риск 1%)

### Что считается и какие файлы

```
compounding_eoy_soy_base_100k/metrics/
  monthly_returns.csv
  full_period_summary.csv

  trades_full_period_summary.csv

fixed_start_100k/metrics/

```

### Схемы CSV

**compounding_eoy_soy_base_100k**

- `monthly_returns.csv`: `year, month, ret_m`  
  `ret_m` — месячная доходность в долях (0.012 = +1.2%) из EoM NAV (UTC+0); месяцы без сделок остаются с `ret_m = 0`.
- `full_period_summary.csv`:  
  `months, cagr, vol_ann, sharpe_ann, sortino_ann, maxdd, mar_full, longest_underwater_months, best_month, worst_month, pos_months_pct, n_trades, hit_rate, profit_factor, avg_win_r, avg_loss_r, payoff_ratio, expectancy_r_mean, expectancy_r_median, std_r, min_r, max_r`
- `yearly_summary.csv`: `year, ret_year, maxdd_year, trades, hit_rate, profit_factor`
- `trades_full_period_summary.csv`:  
  `n_trades, hit_rate, profit_factor, avg_win_r, avg_loss_r, payoff_ratio, expectancy_r_mean, expectancy_r_median, std_r, min_r, max_r`

**fixed_start_100k**

- `monthly_returns.csv`: `year, month, ret_m` (внутри каждого года якорь 100k на 01-01 00:00 UTC−ε)
- `yearly_summary.csv`: `year, ret_year, maxdd_year, trades, hit_rate, profit_factor`
- `trades_full_period_summary.csv`: как у comp (агрегация по всему периоду)

---

### Правила и конвенции

- **Таймзона:** UTC+0.  
- **Шкала:** месячные ретерны из **последнего EoM NAV**; пропуски month‑end — ffill.  
- **Якорь NAV = 100 000 USD.**  
  - **Compounding:** вставляем якорь в `t₀−ε` (до первой точки).  
  - **Fixed:** каждый год — якорь на `YYYY-01-01 00:00 UTC−ε`.  
- **Месяцы без сделок** не удаляем: `ret_m = 0`.  
- **Дисперсии/σ:** выборочные, `ddof = 1`.  
- **R‑метрики (риск 1%):**  
  `R = pnl_pct` (если `pnl_pct` в долях) **или** `R = pnl_% / 100` (если в процентах).  
  `hit_rate` = доля `R > 0`; `PF` = сумма win / |сумма loss| (по **суммам**, не по средним).

---

**R-метрики (риск 1% — STRICT)**  
- База: **1R = 1.0%** капитала на входе.  
- Если `pnl_pct` **в долях** (например, `0.012` = +1.2%) → `R = pnl_pct / 0.01`.  
- Если `pnl_%` **в процентах** (например, `1.2` = +1.2%) → `R = pnl_% / 1.0`.  
- Если в `trades` есть `pnl_r`/`r`, используем **только** если это уже R при риске 1%.

- Вводим эпсилон `eps = 1e-12` для устойчивых сравнений с нулём.  
- **Классификация:**  
  — **win:** `R > +eps`; **loss:** `R < −eps`; нули исключаем из win/loss-групп.  

- **Метрики (по R):**  
  — `hit_rate = share(R > +eps)` (нули не победы);  
  — `profit_factor = sum(R[R>+eps]) / abs(sum(R[R<−eps]))` (**по суммам**, нули не участвуют);  
  — `avg_win_r = mean(R[R>+eps])`, `avg_loss_r = mean(R[R<−eps])`;  
  — `payoff_ratio = avg_win_r / |avg_loss_r|`;  
  — `expectancy_r_mean/median`, `std_r`, `min_r`, `max_r` — по **всем** R как есть (нули включены).

---

### Формулы (institutional‑определения)

- `CAGR = (∏(1 + r_m))^(12/N) − 1`, где `N` — число месяцев.  
- `vol_ann = stdev(r_m, ddof=1) · √12`.  
- `Sharpe_ann = (mean(r_m − rf_m) / stdev(r_m − rf_m, ddof=1)) · √12`, по умолчанию `rf = 0`.  
- `Sortino_ann = (mean(r_m) / stdev(r_m[r_m < 0], ddof=1)) · √12` (downside‑σ только по **отрицательным** месяцам, target=0).  
- **Кривая и просадки (месячная шкала):**  
  `eq_t = ∏(1 + r_m)`, `dd_t = eq_t / cummax(eq) − 1`.  
  `maxdd` = минимум `dd_t` (отрицательное число).  
  `longest_underwater_months` — длина самой длинной серии месяцев, где `dd_t < 0` (месяц восстановления с `dd=0` не включается).  
- **MAR:** `mar_full = cagr / |maxdd|` (по всей месячной кривой).  
- **Годовые (fixed/comp):**  
  `ret_year = ∏(1 + r_m_внутри_года) − 1`,  
  `maxdd_year` — на месячной кривой **внутри года**.

### Что **не** публикуем в этом треке

- Расширенные «institutional»‑метрики (Ulcer, Martin, rolling‑12/36m, Calmar_36m, Kelly, Monte‑Carlo) — **не входят** в 0.7.  
- Сквозные Sharpe/MAR/MaxDD для **fixed** — **не считается** (из‑за ежегодного ресета к 100k).

---

### Мини‑проверки целостности

- Охват: **199 месяцев** (2009‑01 … 2025‑07) — без «дыр».  
- `∏(1 + ret_m)` по comp ≈ `NAV_last / 100000`.  
- `n_trades` в `trades_full_period_summary.csv` = сумма годовых `trades` из `yearly_summary.csv`.  
- PF инвариантен масштабу (совпадает и в долях, и в процентах, если брать **суммы**).

### Нюансы (закрывают частые вопросы)

- **Округления (institutional):**  
  CAGR/vol/maxdd/best/worst — до **6** знаков; Sharpe/Sortino — до **4**; PF/Payoff — до **3**; hit_rate/pos_months — до **6**; R‑метрики — до **6**.  
  - **Последний неполный месяц:** **включаем**; NAV ffill до EoM; месяц остаётся в ряду.  
- **Месяцы без сделок:** остаются с `ret_m = 0` (не выкидывать).  
- **Downside‑σ (Sortino):** считаем **только по отрицательным месяцам** `r_m < 0`, `ddof=1`, target=0.  
- **Sharpe:** `rf=0` (если нет rf‑ряда). Если есть годовой rf → использовать **эффективный** `rf_m = (1 + rf_ann)^(1/12) − 1`.  
- **PF / hit‑rate:** `hit_rate = share(R > 0)` (нули не считаются win). `PF = sum(R>0) / |sum(R<=0)|`; если потерь нет — **PF = inf**.  
- **Longest Underwater:** считаем **инклюзивно** как длину подряд идущих месяцев с `dd < 0`. Месяц восстановления (`dd = 0`) **не входит**.  
- **Санити‑чеки:**  
  `∏(1+ret_m)` (comp) ≈ `NAV_last/100000`;  
  `n_trades(full)` = сумма годовых `trades`;  
  покрытие — **199 месяцев** (2009‑01…2025‑07) без дыр.
