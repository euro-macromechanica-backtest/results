# 📦 Results

> **Важное уточнение:** реализация **охватывает ~10–15% полной логики** и только торговлю относительно **M5‑таймфрейма** полной quant‑стратегии **Euro Macromechanica (EMM)**. Даже при таком охвате результаты демонстрируют **стабильную прибыльность на длинном горизонте при процентном риске от капитала на момент открытия сделки** на минутных данных с наличием **разрывов**. Подробнее см. корневой `README.ru.md`.
>
> - **Cost-модель (включена во все результаты):** доступны 3 варианта результатов:  
>   - **Institutional** *главный трэк* — *$3 за round-turn на 1 стандартный лот (100 000 EUR)*; эквивалент **≈0.3 pip на EURUSD** (≈$1.5 за сторону). Ориентир для институциональных условий; финальная нетто-оценка всегда выполняется на стороне контрагента по его cost-модели (per-million per side, markup, slippage) и на его данных (LP/TCA).
>   - **Retail Advanced** *ритейл-ECN режим использующий сервис кэшбэка на коммиссии.* **$4.5 за round-turn на 1 стандартный лот (100 000 EUR).**
>   - **Retail Baseline Stress** *иллюстрация для оценки при издержках обычного ритейл уровня* — **$7 за round-turn на 1 стандартный лот (100 000 EUR)**, типичный уровень для retail-ECN; эквивалент **≈0.7 pip на EURUSD** (≈$3.5 за сторону). Консервативный «худший случай» с максимально высокими издержками retail-ECN; используется как **стресс-бенчмарк**. 
>   - Механику исполнения ордеров — публичное резюме (*M5 with M1 micro-confirmation*) — см. `strategy_proof/README.ru.md`.
>   - Округление установленно до 5 знаков только при генерации csv, все данные внутри движка стратегии считались без округления. Комиссия вычитывается из каждой сделки после ее закрытия. Капитал после закрытия сделки = прибыль/убыток + комиссия с последней сделки. Следующая сделка открывается установленным процентным риском от капитала - уточнение к точности результатам.

**Доступные результаты по cost model:**
- *Institutional* - риск 1.0% и 1.5% (старт капитала 100 000 usd каждый год и компаунд режим на весь период со старта 100 000 usd)
- *Retail Advanced* - риск 1.0% (старт капитала 100 000 usd каждый год и компаунд режим на весь период со старта 100 000 usd)
- *Retail Baseline Stress* - риск 1.0% (старт капитала 100 000 usd каждый год и компаунд режим на весь период со старта 100 000 usd)

Реализовано ~10–15% полной логики и только 5-минутная торговля с минимально реализованным набором quant-факторов, без институциональных данных (тик-лента, официальные календари, точные TCA). ## 🔐 Верификация целостности

### 1) Проверка **годового манифеста** (перечень входов/выходов года)
- **Linux**
  ```bash
  sha256sum -c results/<cost_model>/<risk>/<mode>/YYYY/artifacts_YYYY.sha256
  ```
- **macOS**
  ```bash
  shasum -a 256 -c results/cost_model/<risk>/<mode>/YYYY/artifacts_YYYY.sha256
  ```
- **Windows (PowerShell)** — прямой эквивалент `-c` отсутствует, проверяйте по строкам из манифеста:
  ```powershell
  Get-Content results\cost_model\<risk>\<mode>\YYYY\artifacts_YYYY.sha256 |
    ForEach-Object {
      $parts = $_ -split '\s+'
      $hash  = $parts[0]
      $file  = $parts[-1]
      (Get-FileHash $file -Algorithm SHA256).Hash.ToLower() -eq $hash
    }
  ```

> Годовые `artifacts_YYYY.sha256` содержат **только хэши (64‑символные SHA‑256)** без путей — это упрощает переносимость и исключает ложные несоответствия из‑за различий каталогов.

### 2) Роллап‑манифест «манифестов» (по треку)
Для удобства аудита у треков присутствуют **роллап‑манифесты** (напр.,  
`integrity/results/headline_2009-2025-07/commission_7usd_per_lot/risk_1p00/fixed_start_100k_manifest/artifacts_2009-2025-07.sha256`)  
— это список **SHA‑256 самих годовых файлов** `artifacts_YYYY.sha256` c **относительными путями**.

**Проверка включения конкретного года в роллап:**  
- **Linux**
  ```bash
  # посчитать хэш файла годового манифеста
  sha256sum results/<track>/<mode>/YYYY/artifacts_YYYY.sha256

  # сопоставить с первой колонкой в роллапе
  grep -F "<ВЫХОД_ХЭША>  results/<cost_model>/<risk>/<mode>/YYYY/artifacts_YYYY.sha256"     integrity/results/<…>/artifacts_2009-2025-07.sha256
  ```
- **macOS**
  ```bash
  shasum -a 256 results/<track>/<mode>/YYYY/artifacts_YYYY.sha256
  grep -F "<ВЫХОД_ХЭША>  results/<cost_model>/<risk>/<mode>/YYYY/artifacts_YYYY.sha256"     integrity/results/<…>/artifacts_2009-2025-07.sha256
  ```
- **Windows (PowerShell)**
  ```powershell
  $h = (Get-FileHash "results\<cost_model>\<risk>\<mode>\YYYYartifacts_YYYY.sha256" -Algorithm SHA256).Hash.ToLower()
  Select-String -Path "integrity\results\<…>artifacts_2009-2025-07.sha256" -SimpleMatch "$h  results\<cost_model>\<risk>\<mode>\YYYYartifacts_YYYY.sha256"
  ```

> Роллап‑манифесты содержат **относительные пути**. Поэтому проверяйте их **из корня репозитория** или приводите пути к тем, что указаны в файле.

## 🖋️ Методология и честность данных

**Методология.** Месячная шкала EoM (NAV на конец месяца, UTC+0). Дисперсии — выборочные (`ddof=1`). **Sortino**: downside-σ только по месяцам `r<0` (target=0). **MAR** (CAGR/|MaxDD|) и **Calmar (36m)** — разведены.  

**Slippage** не моделировался, новости не торгуются (торговля начинается только спустя определенное существенное расстояние движения цены с момента выхода high impact события – когда ликвидность нормализовывается – при определенных quant условиях), UTC+0 с DST, нулевые месяцы включены.

### Метрики на основе месячных ретёрнов (институциональный-стандарт)

**Шкала и данные.**  
- Берутся **месячные ретёрны `r_m`** из **EoM NAV** (последнее значение на конец месяца), **UTC+0**.  
- Перед первой точкой — якорь NAV = **100 000 USD** (`t₀−ε`); пропуски EoM — **ffill**.  
- **Месяцы без сделок включены** (их `ret_m = 0`).  
- Дисперсии — выборочные (`ddof = 1`). Ретёрны — **арифметические**, не лог.

**Аннуализируемые метрики (по `r_m`):**  
- `vol_ann = stdev(r_m) · √12`.  
- `Sharpe_ann = (mean(r_m − rf_m) / stdev(r_m − rf_m)) · √12`, по умолчанию `rf = 0`.  
- `Sortino_ann = (mean(r_m) / stdev(r_m[r_m < 0])) · √12` (downside-σ **только по отрицательным месяцам**, target = 0).  
- `CAGR = (∏(1 + r_m))^(12/N) − 1`, где `N` — число месяцев.

**Просадки и производные:**  
- Месячная кривая: `eq_t = ∏(1 + r_m)`.  
- Просадка: `dd_t = eq_t / cummax(eq) − 1` (отрицательная).  
- **MaxDD** — минимум `dd_t`.  
- **Longest Underwater** — длина самой длинной серии месяцев с `dd_t < 0` (месяц восстановления с `dd=0` **не входит**).  
- **Time Underwater** — доля месяцев с `dd_t < 0`.  
- **MAR (mar_full)** = `CAGR / |MaxDD|`.

**Rolling-метрики:**  
- **Rolling-12m**: `roll_12m_return = ∏(1+r_m_window) − 1`, `roll_12m_sharpe = (mean/σ)·√12` внутри окна (rf=0).  
- **Rolling-36m (Calmar)**:  
  `ret_36m_ann = (∏(1+r_m_window))^(12/36) − 1`,  
  `maxdd_36m` — MaxDD на той же оконной кривой,  
  `calmar_36m = ret_36m_ann / |maxdd_36m|`.  
  > В итоговом отчёте публикуются **MAR (full)**; **Calmar 36m** — в rolling-файле.

**Доп. показатели (если считаются):**  
- **Ulcer Index**: `sqrt(mean( max(0, −dd_t)^2 ))`.  
- **Martin Ratio**: `(mean(r_m)·12) / UlcerIndex`.  
- **Skewness / Kurtosis_excess** — на `r_m` (Fisher).

> Все эти метрики зависят **только от месячных ретёрнов**, не от R.  
> Блок **R-метрик (hit-rate, PF, payoff, expectancy, Kelly)** считается **отдельно** из сделок. 

**R-метрики (STRICT — для всех треков 1% и 1.5%)**  
- База нормировки:
  - **Риск 1%:** `1R = 1.0%` капитала на входе.  
    • если `pnl_pct` в долях (напр. `0.012` = +1.2%) → `R = pnl_pct / 0.01`;  
    • если `pnl_%` в процентах (напр. `1.2` = +1.2%) → `R = pnl_% / 1.0`.  
  - **Риск 1.5%:** `1R = 1.5%` капитала на входе.  
    • если `pnl_pct` в долях → `R = pnl_pct / 0.015`;  
    • если `pnl_%` в процентах → `R = pnl_% / 1.5`.  
  - Если в `trades` есть `pnl_r`/`r`, используем **только** если это уже R при соответствующем риске.

- Эпсилон для устойчивых сравнений: `eps = 1e-12`.

- **Классификация сделок (STRICT):**  
  — **win:** `R > +eps`; **loss:** `R < −eps`; нули исключаем из групп win/loss.

- **Метрики (по R):**  
  — `hit_rate = share(R > +eps)` (нули не считаются победами);  
  — `profit_factor = sum(R[R>+eps]) / abs(sum(R[R<−eps]))` (**по суммам**, нули не участвуют);  
  — `avg_win_r = mean(R[R>+eps])`, `avg_loss_r = mean(R[R<−eps])`;  
  — `payoff_ratio = avg_win_r / |avg_loss_r|`;  
  — `expectancy_r_mean/median`, `std_r`, `min_r`, `max_r` — по **всем** R как есть (нули включены).

_Примечание:_ на исторических данных нулевых R практически нет; STRICT вводится для единообразия и защиты от погрешностей округления.

**Данные с разрывами.** Источник котировок задокументирован; пропуски NAV закрываются `ffill` к каждому EoM. Целостность проверена: **199 месяцев** (2009-01…2025-07) без «дыр»; месяцы без сделок **оставлены** (`ret_m = 0`).

**Поведенческий риск.** Длинные подводные периоды (**Longest UW (Underwater - время ниже хай-водораздела) ≈ 41–42 мес.**) — свойство демо-модели, а не баг. Витрина реализует всего лишь ~**10–15%** логики полной стратегии и применяет **минимальные** квант-фильтры, демонстрируя **стабильную долгосрочную прибыльность** без агрессивной «докрутки». Это удлиняет UW при сохранении умеренной глубины просадок.
